# Task 5: Capture and Analyze Network Traffic Using Wireshark

## Objective
Capture live network packets and identify basic protocols and traffic types using Wireshark.

## Tools Used
- **Wireshark 4.6.6** (free, open-source packet analyzer)
- **Npcap 1.88** (packet capture driver for Windows)
- **Windows Command Prompt** (for generating ICMP traffic via ping)

## Steps Performed

### 1. Installed Wireshark
Downloaded and installed Wireshark 4.6.6 x64 from https://www.wireshark.org. Npcap was installed alongside it to enable packet capture on Windows.

### 2. Started Capture
Selected the **Wi-Fi** interface (active network adapter) and started live capture.

### 3. Generated Network Traffic
- Browsed websites (Google, YouTube) to generate TCP/TLS/DNS traffic
- Ran `ping google.com` in Command Prompt to generate ICMPv6 packets

### 4. Stopped Capture
Stopped capture after approximately 1 minute. Total packets captured: **59,926**

### 5. Filtered Packets by Protocol
Applied display filters in Wireshark to isolate specific protocols:
| Filter | Protocol |
|--------|----------|
| `dns` | DNS |
| `tcp` | TCP |
| `tls` | TLS |
| `http` | HTTP |
| `icmpv6` | ICMPv6 |

### 6. Identified Protocols
At least 5 different protocols were identified in the capture (details in report below).

### 7. Exported Capture File
Saved the capture as `capture_task5.pcap` using File → Save As in Wireshark.

---

## Protocols Identified

| Protocol | Full Name | Purpose |
|----------|-----------|---------|
| **TCP** | Transmission Control Protocol | Reliable data delivery between client and server |
| **DNS** | Domain Name System | Translating domain names to IP addresses |
| **TLS** | Transport Layer Security | Encrypting HTTPS web traffic |
| **HTTP** | Hypertext Transfer Protocol | Unencrypted web requests |
| **ICMPv6** | Internet Control Message Protocol v6 | Network diagnostics (ping) |

---

## Key Findings
- The majority of traffic was **TCP-based**, making up the bulk of 59,926 packets
- **DNS queries** were observed resolving domains like microsoft.com, cloudflare.com, and digicert.com
- **TLS (v1.2 and v1.3)** was used for all HTTPS connections, showing encrypted handshakes (Client Hello, Server Hello, Application Data)
- **HTTP** traffic was also captured showing GET requests to streaming services
- **ICMPv6** Echo Request and Reply packets were generated by the `ping google.com` command

---

## Concepts Learned
- How to use Wireshark to capture live network traffic
- How to apply display filters to isolate specific protocols
- The difference between TCP and UDP, HTTP and HTTPS
- How DNS resolves domain names before a connection is made
- How ICMPv6 is used for network diagnostics (ping)
